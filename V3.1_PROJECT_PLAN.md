当前路基工程施工管理存在以下痛点：
- **数据冗余巨大**: 单公里GB级点云数据
- **精度静态固定**: 无法动态调整
- **图纸提取依赖人工**: 效率低

| 阶段 | 目标 | 指标 |
|------|------|------|
| 短期 | 打通参数提取链路 | 提取时间≤15分钟，准确率≥98% |
| 中期 | 建成Neo4j时空模型 | 冲突响应≤0.3s，准确率≥99.5% |
| 长期 | 施工指导界面 | 冲突提前发现率≥4倍 |

## 三、参数化模型

### 3.1 平曲线参数

| 参数 | 说明 | 示例 |
|------|------|------|
| element_type | 线元类型 | 直线/缓和曲线/圆曲线 |
| start_stake | 起点桩号 | K0+000 |
| end_stake | 终点桩号 | K0+500 |
| azimuth | 方位角(°) | 45.0 |
| R | 半径(m) | 800 |
| A | 回旋参数 | 300 |
| Ls | 缓和长度(m) | 100 |

### 3.2 纵曲线参数

| 参数 | 说明 | 示例 |
|------|------|------|
| stake | 变坡点桩号 | K0+800 |
| elevation | 设计高程(m) | 125.45 |
| grade_in | 入口坡度(‰) | 20.0 |
| grade_out | 出口坡度(‰) | -15.0 |
| curve_length | 竖曲线长度(m) | 200 |

### 3.3 横断面参数

| 参数 | 说明 | 示例 |
|------|------|------|
| normal_width | 路基宽度(m) | 26.0 |
| crown_slope | 路拱坡(%) | 2.0 |
| side_slope | 边坡坡率 | 1:1.5 |
| superelevation | 超高(%) | 8.0 |
| widening | 加宽(m) | 0.8 |

## 四、核心算法

### 4.1 平面坐标计算

```python
# 直线段
x(l) = x₀ + l × cos(ρ)
y(l) = y₀ + l × sin(ρ)

# 圆曲线段
θ(l) = l / R
x(l) = R × sin(θ + ρ₂) - R × sin(ρ₂)
y(l) = R × cos(ρ₂) - R × cos(θ + ρ₂)

# 缓和曲线段 (三次回旋线)
τ = l² / (2 × A²)
x(l) ≈ l × (1 - τ²/10 + τ⁴/216)
y(l) ≈ l³/(6×A²) × (1 - τ²/42)
```

### 4.2 纵断面高程计算

```python
# 抛物线竖曲线
z(x) = z_vp + i₁ × x + (i₂ - i₁) × x² / (2 × L)
```

```
你是公路设计专家。只从图纸中提取参数化设计数据，不要猜测点集。

提取:
- 平面: JD桩号、X/Y坐标、Δ偏角、R半径、A回旋参数、Ls缓和长
- 纵断: 变坡点桩号、高程、i1/i2纵坡(‰)、竖曲线L
- 横断: 宽度、路拱坡、边坡率、超高值

输出JSON，每个字段附置信度(0-1)。
```
